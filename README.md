# term-project-
#(1)程式的原理與功能
  本程式主要目地為儲存一些函式在functions.py，藉由這些函式來產生圖片，其中有2D、3D、4D圖(4D圖用切片等高線圖表示)
  原理為藉由keys.py(python 的dictionary功能來聯繫各個python檔(由使用者輸入的代號得到一串代表函式的字串，再用getattr得到函式)，最後由plotfunc.py來處理畫圖，並由main.py架構主要程式(負責與使用者互動)
  功能:2D有畫圖功能以及有使用者輸入區間的函數平均值
       3D畫圖功能以及剖面圖功能
       4D剖面等高線圖
       
#(2)使用方式:
  第一步:輸入想要的函數維度(2-4)
  第二步:選擇要使用的函數(輸入代號，代號對應的內容會在終端機告知)
  第三步:輸入x、y的範圍(目前只有做到單一連續的範圍(給使用者輸入initial value及final value))
  第四步:輸入想要的點的數量(linspace的分割數量)
  第五步:若是3D圖得到第一張圖後會詢問是否要看剖面圖
        第一步:輸入想要固定的軸(x or y)
        第二步:輸入要固定的值(請輸入在前面的定義域範圍內)
        
#(3)程式的架構:
  分為四個part:
  functions.py負責儲存所有函數(用於計算、儲存函數字串)
  keys.py負責儲存dictionary及提取使用者所輸入的key所對應的value
  plotfunc.py藉由keys.py連結functions.py，負責繪圖及平均值的計算
  main.py主程式，主要的輸出、使用函式及詢問過程都在這邊
  
#(4)開發過程:
  先想要放上什麼常見的函數，因此先從functions.py開始，製作完2D函數之後，開始尋找3D的函數，例如上課用到的van de walls 以及 gravity、wave，4D函數我能想到的只有gravity，因此最後只放上gravity
  接著，因為一開始的構思就是兩個程式作為library但為了方便使用者輸入，因此我多加了一個keys.py，也使我方便連結兩個主要的library與主程式
  因此完成functions 的架設之後開始寫keys對應的value，為了方便我直接讓value跟我functions裡的函數名稱一樣，但我不知道提取出來的字串如何讓它變成函數名稱，因此問了perplexity，他給出了一個答案:getettr
  也就是keys得到了一個字串fname，我就可以用func=getattr(functions,fname)這樣就讓fname從字串變成可以用的函數，直接用func(x)就可以用functions.py的函數了
  接下來要做的就是畫圖，因此我把繪圖函數與數學函數分開，獨立出了plotfunc.py，這個過程沒有什麼太困難的過程，不過對於4D圖形我該怎麼畫我想了很久，最後上網找到     (https://stackoverflow.com/questions/9960456/matlab-need-to-make-a-4d-plot-3d-colour-color)    ，他是用顏色的方法，跟上課學到的等高線一樣，但出現了一個問題3D圖上無法清楚呈現等高線，因此我最吼的想法就是'切片'，給使用者選擇3個z值，分別輸出三張等高線圖的slice。
    最後，也是最重要的就是主程式，過程中沒有什麼太大的困難，一切照原本想好的寫下去
    但整段過程有一個很花時間的事情就是在所有程式完成時的debug，我從2D的每個函數開始試，遇到的問題有:input在def的函示內，而x是一個陣列，因此在終端機會一直要求我輸入，最後決定把計算的函式分開，寫一個負責計算，一個只負責輸出函數(用字串輸出)，負責計算的在用eval(負責輸出的函數輸出的字串)來計算
    在3D遇到的困難式矩陣上的困難，有時不知道為什麼可能x就變成一階張量了，最後輸出的圖從上方看下去就是一條直線(x,y)沒有用到np.meshgrid，總之一值遇到這樣的情況，最後一個個式都改完整了  
    而4D意外的沒有任何bug試一次就成功了(但它的顏色對應圖例位置跑掉)，後來改一下就完整了
